## Backlogs

Un backlog es una lista de tareas pendientes que se deben completar en un proyecto de desarrollo de software. 
En la metodología Scrum, el backlog es una herramienta utilizada para organizar y priorizar las tareas a realizar durante el proyecto. 
Se utiliza para identificar y definir claramente qué tareas son necesarias para completar el producto final y para establecer un plan para completarlas.

El backlog es mantenido y actualizado por el equipo de desarrollo y el Scrum Master a lo largo del proyecto. 
Las tareas en el backlog pueden incluir desde requisitos detallados hasta tareas generales. 
El backlog también puede incluir información sobre la importancia de cada tarea y el tiempo estimado para completarla. 
Durante cada sprint, el equipo selecciona las tareas más importantes del backlog y trabaja en ellas hasta completarlas.
